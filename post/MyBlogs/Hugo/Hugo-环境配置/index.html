Hugo-环境配置

Mon, Aug 26, 2019

安装Hugo

在官网下载Windows版本的Hugo安装包，我下载的版本是hugo_0.57.0_Windows-64bit.zip。

解压到目录，比如D:\MyHugo，并将该路径添加到“环境变量”中。

设置“环境变量”之后，在cmd下执行命令，如果输出类似于如下所示的内容，则表明安装成功。

D:\MyHugo> hugo version
Hugo Static Site Generator v0.57.0-9B00E647 windows/amd64 BuildDate: 2019-08-14T
08:05:40Z

初始化Blog

找到一个目录，用于存放Hugo生成的站点，我这里直接在D:\MyHugo目录下生成站点。

D:\MyHugo> hugo new site mysite

成功的话，会生成D:\MyHugo\mysite文件夹，里面包含一系列目录和配置文件，我们简单介绍一下：

<DIR>          archetypes
            82 config.toml
<DIR>          content
<DIR>          data
<DIR>          layouts
<DIR>          resources
<DIR>          static
<DIR>          themes
  • config.toml: 是网站的配置文件，这是一个TOML文件，全称是Tom’s Obvious, Minimal Language，这是它的作者GitHub联合创始人Tom Preston-Werner 觉得YAML不够优雅，创建出来的一个新格式。如果你不喜欢这种格式，你可以将config.toml替换为YAML格式的config.yaml，或者JSON格式的config.json，Hugo都支持。
  • archetypes: 该目录
  • content: 该目录里放的是Markdown文章。
  • data: 该目录用来存放数据文件，一般是JSON文件，Hugo提供了相关命令可以从data目录下读取相关的文件数据，然后渲染到HTML页面中，将业务数据与模板分离。
  • layouts: 该目录里放的是网站的自定义模板文件，Hugo优先使用layouts目录下的模板，未发现再去themes目录下查找。
  • static: 该目录里放的是一些图片、css、js等静态文件。Hugo在渲染时，会直接将static目录下的文件直接复制到public目录下，不会做任何渲染。
  • themes: 该目录存放的是网站模板主题。可以下载多个主题，themes目录下的每个子目录代表了一个主题，可以通过在config.toml中通过参数theme指定主题，即themes目录下的子目录名字， 也可以在执行hugo命令渲染时通过增加flag参数–theme=xxx指定。

创建页面与文章

进入mysite目录，创建一个页面，Hugo会在content目录下生成一个about.md文件。

D:\MyHugo\mysite> hugo new about.md

如果是博客日志，最好将md文件放在content的post目录里，Hugo会在content\post目录下生成一个first.md文件。

D:\MyHugo\mysite> hugo new post/first.md

现在，站点的基本架构已经搭好了，还缺一个模板。Hugo默认生成的网站是没有任何皮肤模板的，要想站点正常显示，你还需要下载一个主题模板。

安裝主題

首先进入到themes目录下，然后clone一个模板

D:\MyHugo\mysite> cd themes
D:\MyHugo\mysite\themes> git clone https://github.com/spf13/hyde.git

本地调试

启动本地调试：

D:\MyHugo\mysite> hugo server --theme=hyde --buildDrafts --watch
  • --theme: 指定主题模板
  • --buildDrafts: 将草稿也一同编译。
  • -–watch: 或者-w选项打开的话，将会监控到文章的改动从而自动去刷新浏览器，不需要自己手工去刷新浏览器，非常方便。

浏览器里打开如下网址，即可浏览效果。

http://127.0.0.1:1313

部署到Github Pages

如果需要部署在 GitHub Pages 上，首先在GitHub上创建一个Repository，命名为：zhannnnn5g.github.io （zhannnnn5g替换为你的Github用户名）。

然后，在D:\MyHugo\mysite目录下，将该仓库下载到本地来（由于本地管理多个SSH，且该Github账号的Host被设置为zhannnnn5g，因此将git@后紧跟的github.com改为zhannnnn5g），并将目录名改为public：

D:\MyHugo\mysite> git clone git@zhannnnn5g:zhannnnn5g/zhannnnn5g.github.io.git public

下载下来后，进入public目录，配置仓库级的user和email：

$ cd public
$ git config  user.email "你的github邮箱地址"
$ git config  user.name "zhannnnn5g"

然后，退出public目录，在站点根目录（D:\MyHugo\mysite）下执行 hugo 命令生成最终页面：

D:\MyHugo\mysite> hugo --theme=hyde --baseUrl="http://zhannnnn5g.github.io/"

如果一切顺利，所有静态页面都会生成到public目录，这样，public目录下的所有文件都归在Git的管理之下了。这里有一点要注意，如果你的所有文章的文章头都设置为draft: true，那么Hugo会将其略过，不会为你生成文章。这样你上传到Github后，只会显示一个首页，没有文章页。

然后将pubilc目录里所有文件push到Github仓库的 master 分支。

D:\MyHugo\mysite\public> git status
D:\MyHugo\mysite\public> git add .
D:\MyHugo\mysite\public> git commit -m "initial commit for hugo blog"
D:\MyHugo\mysite\public> git push origin master

然后，在浏览器里访问：http://zhannnnn5g.github.io/
